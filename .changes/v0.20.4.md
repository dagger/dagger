## v0.20.4 - 2026-04-08

### Added
- Add experimental `dagger up` to start module-defined services on the host, including `@up` annotations, workspace/module `services()` APIs, and configurable port mappings by Yves Brissaud in https://github.com/dagger/dagger/pull/11959
- Add `@cache` directive for module function cache control by Alex Suraci in https://github.com/dagger/dagger/pull/12920
- Add `--failfast` to `dagger check` and `WithFailFast` in SDKs by @kpenfound in https://github.com/dagger/dagger/pull/11956
- Add support to set primitive values and secret arrays in user defaults by @marcosnils in https://github.com/dagger/dagger/pull/12898
- Add `Changeset.diffStats()` and structured `DiffStat` entries for renames and line-count summaries by @grouville + @tiborvass in https://github.com/dagger/dagger/pull/12002
- Add support for `Directory.chown()` with usernames and group names by @alexcb in https://github.com/dagger/dagger/pull/12128
- Add Dockerfile `ADD --unpack` support by @tiborvass + @grouville in https://github.com/dagger/dagger/pull/12096
- Add Go toolchain tag support for Dockerfile frontend features such as `COPY --exclude` and `COPY --parents` by @alexcb in https://github.com/dagger/dagger/pull/12896
- Dang SDK: add enum/interface/scalar support plus `@check` and `@generate` directives by Alex Suraci in https://github.com/dagger/dagger/pull/12830
- Dang SDK: add `@up` annotation support by Yves Brissaud in https://github.com/dagger/dagger/pull/12919
- Elixir SDK: support `Enum` types by @Nero-F in https://github.com/dagger/dagger/pull/12862
- Go SDK: write dependency bindings to separate `internal/dagger/<dep>.gen.go` files by @TomChv in https://github.com/dagger/dagger/pull/11962
- PHP SDK: add support for implicit getters by @charjr in https://github.com/dagger/dagger/pull/11689
- Rust SDK: add a builder for `Config` by @fdiakh in https://github.com/dagger/dagger/pull/12832

### Changed
- Make `dagger connect` faster by reducing driver probing and setup work before command execution by @tiborvass in https://github.com/dagger/dagger/pull/11769
- Expand workspace support with workspace-scoped `checks()` and `generators()` APIs, richer workspace metadata, and clearer path/boundary semantics by @shykes in https://github.com/dagger/dagger/pull/11995
- `dagger check` and `dagger generate` now resolve against the current workspace rather than only the current module by @shykes in https://github.com/dagger/dagger/pull/11995
- Stop expanding literal dotenv values and arguments so literal strings remain literal by @marcosnils in https://github.com/dagger/dagger/pull/11957
- Go SDK: pin generated modules to the matching dagger-go-sdk commit by @TomChv in https://github.com/dagger/dagger/pull/11826

### Deprecated
- Set secrets via dotenv segments without a scheme is deprecated and will require an explicit scheme in a future release by @marcosnils in https://github.com/dagger/dagger/pull/11957

### Fixed
- Fix Dockerfile `RUN` heredoc failing with `command not found` (exit 127) by preserving `/dev/pipes/` mounts in the OCI spec by @majiayu000 in https://github.com/dagger/dagger/pull/12020
- Fix `dagger init` crashing when `.env` exists as a directory by @paikend in https://github.com/dagger/dagger/pull/12868
- Fix `WithExec` with `Expand=true` so environment variables are also expanded in `RedirectStdout`, `RedirectStderr`, and `RedirectStdin` paths by @shykes in https://github.com/dagger/dagger/pull/12846
- Fix Helm chart support for custom liveness/readiness probes by @shykes in https://github.com/dagger/dagger/pull/12859
- Fix Helm chart `envFrom.secretRef` indentation in the engine StatefulSet by @fixeasy in https://github.com/dagger/dagger/pull/12906
- Fix `dagger shell` object-argument errors to show a clear message when a string literal is passed instead of an object ID by @shykes in https://github.com/dagger/dagger/pull/12842
- Fix lazy image blob handling so container results remain local after `ContainerDagOp` returns by @marcosnils in https://github.com/dagger/dagger/pull/12861
- Fix .env user defaults being silently ignored when a constructor arg has a schema default value by @shykes in https://github.com/dagger/dagger/pull/12854
- Dang SDK: fix workspace arguments and filesync support to match `/dagger-sdk` behavior by Alex Suraci in https://github.com/dagger/dagger/pull/12830
- Python SDK: fix `Config.log_output` handling for in-memory streams such as `StringIO` by @paikend in https://github.com/dagger/dagger/pull/12867
- Rust SDK: fix timeouts and default `Config` behavior, and improve GraphQL error parsing by @fdiakh in https://github.com/dagger/dagger/pull/12832
