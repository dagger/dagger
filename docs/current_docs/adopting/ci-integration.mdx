# CI Integration

## Cloud Checks

The standard way to run Dagger in CI is [Cloud Checks](https://dagger.cloud) — managed CI with no runners to configure:

1. `dagger login`
2. Connect your repository in the Dagger Cloud dashboard
3. Push code — checks run automatically and report as commit statuses on your PRs

Cloud Checks runs on managed infrastructure with integrated compute and caching. No self-hosted runners, no CI configuration files, no maintenance.

## Hybrid mode

If you're evaluating Dagger incrementally or need to run alongside an existing CI system, you can call Dagger from any CI platform that supports containers:

```yaml
# GitHub Actions
- run: dagger check
```

This is useful as a bridge while migrating to Cloud Checks. Because Dagger runs identically everywhere, you can start with hybrid mode and switch to Cloud Checks when ready — your checks don't change.
