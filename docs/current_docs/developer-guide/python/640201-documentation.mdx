# Inline Documentation


Dagger Modules and Dagger Functions should be documented so that descriptions are shown
in the API and the CLI - for example, when calling `dagger functions` and `dagger call --help`.

## Documentation strings

The following code snippet shows how to use Python's
[documentation string conventions][docstrings] for adding descriptions to:
- The whole [module][pymod] or [package][pypkg]
- Object type classes (group of functions)
- Function methods

<!-- loud_hello has a multi-line docstring on purpose -->
```python file=./snippets/documentation/docstrings/main.py
```

:::note
To use multiple Python source files, the `src/main.py` module needs to be converted
into a [package][pypkg]: `src/main/__init__.py`. In this case, the Dagger Module
is documented at the top of the `__init__.py` file, just like any Python package.
:::
