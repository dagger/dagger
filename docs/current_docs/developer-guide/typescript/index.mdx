# Developing with TypeScript

Welcome to the TypeScript developer guide!

This guide is intended for TypeScript developers who want to add custom functionality to Dagger, either by creating a new Dagger Module or extending an existing one.

If you're just getting started, the learning path below will give you the essential information you need to understand and begin developing Dagger Modules in TypeScript.

- Understand the [difference between Dagger Modules and Dagger Functions](../overview/382910-modules-vs-functions.mdx)
- Learn about the [architecture](../overview/482011-architecture.mdx) and [execution environment](../overview/942201-execution-environment.mdx) for Dagger Modules
- Write [your first Dagger Module in TypeScript](./506316-first-module.mdx)
- Understand [arguments and return values](./292372-functions.mdx)
- Document your module with [inline comments](./104624-documentation.mdx)
- Streamline your module with a [constructor](./520201-constructor.mdx)
- Work with [secrets](./527581-secrets.mdx)
- Call code from [other modules](./232836-dependencies.mdx)
- [Handle errors](./658262-error-handling.mdx) and [debug](./372032-debugging.mdx) your module
- Get [IDE support](./629502-ide-integration.mdx) for your module development

Detailed [reference documentation for the TypeScript SDK](../../reference/typescript/modules.md) is also available.
