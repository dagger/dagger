# Developing Modules

When pre-built modules don't cover your needs, you can write your own. A module is a package of functions — written in any supported SDK — that extends the Dagger API.

## When to develop a module

Modules range from project-specific to general-purpose.

A project-specific module automates something unique to your codebase — a custom deploy script, a build that stitches together internal tools, an integration test that needs your exact infrastructure. Start here. It lives in `.dagger/modules/`, it doesn't need to be polished, and you can always generalize later.

A general-purpose module solves a problem for anyone — linting, testing, container builds. If you're building one of these, check what's already installable with `dagger install` before starting from scratch.

## Get started

**[Start with the Base Edition](/extending/editions/dang)** — uses Dang, Dagger's native DSL. No codegen, no external dependencies. The recommended starting point.
