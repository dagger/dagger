# Developing Modules

When pre-built modules don't cover your needs, you can write your own. A module is a package of functions — written in any supported SDK — that extends the Dagger API.

## When to develop a module

**Install first.** Browse the [Daggerverse](https://daggerverse.dev) and check if someone has already built what you need. `dagger install` is always faster than writing code.

**Then customize.** If an existing module is close but not quite right, you can often configure it via your workspace config rather than forking it.

**Then build.** Write a module when your workflow is specific to your project, you need to compose multiple tools, or you want to share reusable automation with the community.

## Pick your language

Each edition is a complete, self-contained guide to developing modules in that language:

- **[Dang Edition](/extending/editions/dang)** — Dagger's native DSL. No codegen, no external dependencies. The fastest path.
- **[Go Edition](/extending/editions/go)** — For performance, systems work, and existing Go codebases.
- **[TypeScript Edition](/extending/editions/typescript)** — For web toolchains and existing Node.js tools.
- **[Python Edition](/extending/editions/python)** — For scripting, data workflows, and existing Python tools.
