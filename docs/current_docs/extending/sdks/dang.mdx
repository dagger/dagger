# Dang SDK Docs Outline

Dang is Dagger's native DSL. Its docs should teach Dang as a real SDK, not as universal pseudocode for every module developer.

## Proposed Structure

1. Start with a Dang module.
2. Understand `main.dang` and module metadata.
3. Define types, functions, arguments, and return values.
4. Use documentation strings and directives.
5. Work with core Dagger types.
6. Add module dependencies.
7. Use default paths, ignore patterns, checks, and generators.
8. Test Dang modules.
9. Troubleshoot parser, type, and runtime failures.
10. Release and maintain Dang modules.
