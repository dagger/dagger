# TypeScript SDK Docs Outline

The TypeScript SDK docs should teach module developers the Dagger model through TypeScript decorators, runtime choices, package management, and editor workflows.

## Proposed Structure

1. Start with a TypeScript module.
2. Choose Node, Bun, or Deno.
3. Understand generated SDK files and `tsconfig.json`.
4. Define objects, functions, arguments, and return types.
5. Document module APIs with decorators and comments.
6. Work with core Dagger types.
7. Add module dependencies.
8. Use default paths, ignore patterns, checks, and generators.
9. Test TypeScript modules.
10. Manage package dependencies and lockfiles.
11. Troubleshoot runtime and codegen failures.
12. Release and maintain TypeScript modules.
