# Checks

Checks are automated validations that run on every commit to ensure code quality, security, and standards compliance. They're the primary way Dagger helps you maintain quality in your codebase.

**A check is a function that validates your code without requiring any arguments.** This means checks can run anywhere — locally before you commit, in CI on every push, or before deploying to production — with a single `dagger check` command.

## Running Checks

```shell
# Run all checks concurrently
dagger check

# List available checks
dagger check -l

# Run specific checks using patterns
dagger check lint-*
dagger check security-scan
```

Checks run in parallel for maximum performance. The command exits with a non-zero status if any check fails:

```yaml
# Example GitHub Actions
- name: Run checks
  run: dagger check
```

## Checks from Installed Modules

The easiest way to add checks to your project is by installing modules that provide them:

```shell
# Install modules that provide checks
dagger install github.com/dagger/eslint
dagger install github.com/dagger/vitest

# List all checks
dagger check -l

# Example output:
# eslint:lint
# vitest:test
```

Checks are namespaced with the module name:

```shell
# Run all checks from a specific module
dagger check eslint:*

# Run a specific check
dagger check vitest:test
```

## Filtering Checks

Use glob patterns to selectively run checks:

```shell
# Run all lint checks
dagger check lint-*

# Run security-related checks
dagger check security-*

# Run a specific check
dagger check code-style
```

## Creating Checks

To create a check, mark a Dagger Function with the check annotation. Checks must not require any arguments — they should validate the current state of the repository. Checks can accept **optional** arguments to customize behavior.

<Tabs groupId="language" queryString="sdk">
<TabItem value="go" label="Go">

```go
// Runs linting checks on the code
// +check
func (m *MyModule) LintCode(ctx context.Context) error {
    _, err := dag.Container().
        From("golangci/golangci-lint:latest").
        WithExec([]string{"golangci-lint", "run"}).
        Sync(ctx)
    return err
}
```

</TabItem>

@function
