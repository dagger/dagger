# Modules

Modules are packages of functions that add capabilities to your workspace. You can install pre-built modules from the community or write your own.

## Using Modules

Install a module into your workspace:

```shell
dagger mod install github.com/dagger/eslint
```

See what functions it provides:

```shell
dagger call eslint --help
```

Call its functions:

```shell
dagger call eslint lint
```

## Where Modules Come From

Modules can be installed from:

- **GitHub**: `dagger mod install github.com/dagger/eslint`
- **Local paths**: `dagger mod install ./my-module`
- **Any Git URL**: `dagger mod install github.com/org/repo/path@v1.0`

## Module vs Workspace

A module (`dagger.json`) is a package of code — functions, types, and dependencies. A workspace is your Git repository. Its configuration (`.dagger/config.toml`) declares which modules to use and how to configure them.

You interact with modules through your workspace:

```shell
# Add a module to your workspace
dagger mod install github.com/dagger/vitest

# Configure it
# Edit .dagger/config.toml to add [modules.vitest.settings]

# Use it
dagger call vitest test
dagger check
```

## Developing Your Own Modules

When pre-built modules don't cover your needs, you can write your own:

```shell
mkdir -p .dagger/modules/my-ci
```

Add a `dagger.json` file in that directory to declare the module's name, SDK, and dependencies. Then install it into the workspace with `dagger mod install ./.dagger/modules/my-ci --name my-ci` so it can be used alongside other workspace modules.

See the [Module Developer Guide](/extending) for the full guide.
