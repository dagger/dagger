# Workspaces

A workspace is your Git repository. Dagger stores its configuration in a `.dagger/` directory, created automatically when you first install a module.

## Configuration

The `.dagger/config.toml` file declares which modules are installed and their configuration.

```toml
# .dagger/config.toml

[modules.eslint]
source = "github.com/dagger/eslint"

[modules.vitest]
source = "github.com/dagger/vitest"

[modules.prettier]
source = "github.com/dagger/prettier"
```

The `.dagger/` directory also contains:
- **`lock`** — Pins module versions for reproducible builds
- **`modules/`** — Local modules you develop as part of your project

## Installing Modules

Add modules to your workspace with `dagger install`:

```shell
dagger install github.com/dagger/eslint
dagger install github.com/dagger/vitest --name tests
```

Each module becomes available under its name:

```shell
dagger call eslint lint
dagger call tests test
```

## Configuring Modules

Modules can be configured in your workspace config. Configuration values are passed as constructor arguments to the module:

```toml
[modules.go]
source = "github.com/dagger/go"

[modules.go.config]
goVersion = "1.22"
```

## Version Control

The `.dagger/` directory should be committed to version control so your team shares the same configuration.
