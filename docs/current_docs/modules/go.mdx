# Go

Official module: [dagger/go](https://github.com/dagger/go)

Install with `dagger mod install github.com/dagger/go`.

## What It Is

The Go module is for Go workspaces that want one shared way to test, lint, and run `go generate`.

It scans the workspace for `go.mod` files, treats each one as a Go module, and gives you workspace-level functions that run across all of them. That makes it a good fit for monorepos, service repos, and libraries with generated code.

## Use It For

- Test every Go module with the same command.
- Run `golangci-lint` across the workspace.
- Run `go generate` before a build or PR.
- Skip tests, lint, or generation in selected modules with marker files.

## Main Workflows

| Workflow | Function | Use it for |
| --- | --- |
| Test | `test-all` | Check that every discovered Go module passes tests. |
| Lint | `lint-all` | Check that every discovered Go module passes `golangci-lint`. |
| Generate | `generate-all` | Run `go generate` everywhere generated files are part of the project. |

## How to Use It

Start with `test-all`. That is the simplest proof that the module sees your Go workspace correctly.

Add `lint-all` when the project has a lint policy. Add `generate-all` when generated code, embedded assets, mocks, protobuf output, or other generated files are part of normal development.

The module also exposes discovery helpers for workflows that need to inspect the Go module layout. Most users should not start there. Start with the checks.

## Workspace Fit

Use this module when the repo contains one or more Go modules. It is especially useful when several Go modules should share the same Go version and the same CI checks.

If one module should not run a workflow, use the module's skip marker settings instead of splitting the repo into separate check systems.
