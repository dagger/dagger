# Go

Official module: [dagger/go](https://github.com/dagger/go)

Install with `dagger mod install github.com/dagger/go`.

## What It Is

The Go module is the official Dagger toolchain for Go workspaces. It discovers Go modules, runs tests, runs generation, and runs lint across the workspace.

## Use It For

- Running Go tests
- Building Go binaries
- Managing Go toolchain versions
- Producing artifacts for CI and release workflows

## Functions

| Function | Purpose |
| --- | --- |
| `test-all` | Run tests in every discovered Go module. |
| `lint-all` | Run `golangci-lint` in every discovered Go module. |
| `generate-all` | Run `go generate` in every discovered Go module. |
| `modules` | Return every Go module discovered from workspace `go.mod` files. |
| `module` | Return the Go module containing a workspace path. |
| `version` | Go version used for test and generate commands. |
| `include-extra-files` | Extra workspace-root include patterns mounted for each module's Go commands. |
| `skip-test-filename` | Marker filename that skips tests at or above a Go module root. |
| `skip-lint-filename` | Marker filename that skips lint at or above a Go module root. |
| `skip-generate-filename` | Marker filename that skips generation at or above a Go module root. |

## How to Read This API

Start with `test-all`, `lint-all`, and `generate-all`. Those are the workspace-level actions.

Use `modules` and `module` when a workflow needs to inspect the Go module layout.

Use the skip marker functions to document or configure per-module opt-outs.
