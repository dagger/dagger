# Modules

Official modules give a workspace useful work to do.

Install a module when you want to add a real tool to your project: test Go code, run Pytest, lint shell scripts, format source, or validate Helm charts. Most modules are useful because they expose checks, generators, or repair workflows that can run the same way locally, in CI, and in Dagger Cloud.

Each page answers three questions:

1. What is this module for?
2. Which checks or generators matter first?
3. How does it fit into a workspace?
