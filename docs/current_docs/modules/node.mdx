# Node

Official module: [dagger/node](https://github.com/dagger/node)

Install with `dagger mod install github.com/dagger/node`.

## What It Is

The Node module prepares a Node container with your source and dependencies.

It is a foundation module. It does not define a check or generator by itself. Use it when your own module or workflow needs a predictable Node environment, then pair it with a tool module such as ESLint, Prettier, Jest, or Vitest.

## Use It For

- Install dependencies once and reuse the result.
- Build JavaScript or TypeScript projects.
- Give other modules the same Node image, source directory, and package manager.
- Create custom workflows when an official tool module is not enough.

## Main Workflows

- `base`: get a Node container ready for project work.
- `with-install-all`: install the project's dependencies.
- `with-install-package`: add one package before running a custom workflow.
- `with-install-local-package`: use a local package directory as part of the workflow.

## How to Use It

Node is not usually something you run with `dagger check` or `dagger generate` by itself. Use it to prepare the environment that another module uses.

For checks, install the specific tool module:

- ESLint for linting
- Prettier or Biome for formatting
- Jest or Vitest for tests

For custom workflows, use Node directly when you need a build command, package install step, or base layer for your own module.

Workspace settings can control the project source directory, package manager, and base image.

## Workspace Fit

This module belongs in JavaScript and TypeScript workspaces that need shared dependency setup. It keeps runtime setup separate from checks, so each tool module can stay focused.
