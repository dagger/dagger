# Node

Official module: [dagger/node](https://github.com/dagger/node)

Install with `dagger mod install github.com/dagger/node`.

## What It Is

The Node module is the base toolchain for JavaScript and TypeScript workspaces. Other modules can build on it, and users can call it directly when they need a Node container with dependencies installed.

## Use It For

- Installing dependencies
- Running package scripts
- Building frontend or backend projects
- Sharing Node setup across other JS modules

## Functions

| Function | Purpose |
| --- | --- |
| `base` | Runtime container for Node. |
| `base-image-address` | Base image to use. |
| `package-manager` | Package manager to use. |
| `source` | Source directory for the project. |
| `with-install-all` | Install all required packages. |
| `with-install-package` | Install one package. |
| `with-install-local-package` | Install a local package from a `Directory`. |

## How to Read This API

Start with `base` when you need a container. Use the install functions before a build, test, lint, or format step.

This module is also a foundation for `eslint`, `prettier`, `jest`, and `vitest`.

## Checks

This module is mostly a runtime and dependency toolchain. Checks usually belong in modules that run a specific tool, such as ESLint, Jest, Vitest, or Prettier.
