# Vitest

Official module: [dagger/vitest](https://github.com/dagger/vitest)

Install with `dagger mod install github.com/dagger/vitest`.

## What It Is

The Vitest module runs Vitest tests for JavaScript and TypeScript projects.

Use it when the project uses Vitest, especially Vite apps and modern frontend packages.

## Use It For

- Run Vitest as a workspace check.
- Validate frontend packages and libraries.
- List discovered tests when test selection is unclear.

## Main Workflows

- `test`: run the Vitest suite.
- `list`: show which tests Vitest sees.

## How to Use It

Install the module, then run `dagger check`. The check should run the Vitest suite for the selected source directory.

Use `list` when you need to understand what the runner found before it executes.

Workspace settings can control the source directory, package manager, and Node base image.

Vitest is a good check for packages where feedback speed matters. Pair it with ESLint, Prettier, or Biome for code quality checks.

## Workspace Fit

Use this module when Vitest is the test runner. Use Jest instead for projects that already depend on Jest conventions and config.
