# Quickstart

## Explore a Dagger module

Once you've installed the Golang module, start exploring it to see what you can do with it. There are various ways to do this:

- Read the [module's auto-generated documentation](https://daggerverse.dev/mod/github.com/kpenfound/dagger-modules/golang@44564d6443a919a2c3f1dd7ad02fc22b5fb9db8c) in the Daggerverse
- Inspect the [module's source code](https://github.com/kpenfound/dagger-modules/blob/44564d6443a919a2c3f1dd7ad02fc22b5fb9db8c/golang/main.go)
- Use the Dagger CLI to inspect the module

Let's use the Dagger CLI to list the Dagger Functions available in the Golang module:

```shell
dagger -m golang functions
```

You should see the following output:

```
Name              Description
base              Sets up the Container with a golang image and cache volumes
build             Build the Go project
build-container   Build a Go project returning a Container containing the build
build-remote      Build a remote git repo
container         The go build container
golangci-lint     Lint the Go project
project           The go project directory
test              Test the Go project
with-container    Bring your own container
with-project      Specify the Project to use in the module
```

From the output, you can see that this Golang module includes Dagger Functions to lint (`lint`), test (`test`), build (`build`), and package (`buildContainer') a Go application.

Dagger Functions, just like regular functions, can accept arguments. These might be simple string, numeric or Boolean arguments, or more complex Dagger-specific types like `Directory`, `File`, or `Container`.

You can use the Dagger CLI to dynamically inspect a Dagger Function's arguments and display a context-sensitive list of its supported arguments and sub-commands. Use the following command to list the available arguments for the `test` Dagger Function:

```shell
dagger -m golang call test --help
```

You should see the following output:

```
Test the Go project

Usage:
  dagger call test [flags]

Flags:
      --args go test       Arguments to go test
                           +default "./..."
      --source Directory   The Go source code to test

Global Flags:
  -d, --debug             show debug logs and full verbosity
      --json              Present result as JSON
  -m, --mod string        Path to dagger.json config file for the module or a directory containing that
                          file. Either local path (e.g. "/path/to/some/dir") or a github repo (e.g.
                          "github.com/dagger/dagger/path/to/some/subdir")
  -o, --output string     Path in the host to save the result to
      --progress string   progress output format (auto, plain, tty) (default "auto")
  -s, --silent            disable terminal UI and progress output
  -v, --verbose count     increase verbosity (use -vv or -vvv for more)
```

Note the `args` and `source` flags. You'll use them in the next section!
