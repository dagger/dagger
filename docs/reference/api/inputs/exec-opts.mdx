Additional options for executing a command

```graphql
input ExecOpts {
  stdin: String
  redirectStdout: String
  redirectStderr: String
}
```


### Fields

#### [<code style={{ fontWeight: 'normal' }}>ExecOpts.<b>stdin</b></code>](#)<Bullet />[`String`](/reference/api/scalars/string) <Badge class="secondary" text="scalar"/>
> Optionally write to the command's standard input

- Null means don't touch stdin (no redirection)
- Empty string means inject zero bytes to stdin, then send EOF
> 

#### [<code style={{ fontWeight: 'normal' }}>ExecOpts.<b>redirectStdout</b></code>](#)<Bullet />[`String`](/reference/api/scalars/string) <Badge class="secondary" text="scalar"/>
> Optionally redirect the command's standard output to a file in the container.
Null means discard output.
> 

#### [<code style={{ fontWeight: 'normal' }}>ExecOpts.<b>redirectStderr</b></code>](#)<Bullet />[`String`](/reference/api/scalars/string) <Badge class="secondary" text="scalar"/>
> Optionally redirect the command's standard error to a file in the container.
Null means discard output.
>
