# Developing Modules

A module is a package of Dagger functions. To write one, start by choosing an SDK.

## Choose an SDK

- [Base Edition](/1.0.0-beta.2/extending/editions/dang): Dang. Smallest setup; best for pure Dagger orchestration.
- [Go Edition](/1.0.0-beta.2/extending/editions/go): Go. Strong types, compiled tools, Go libraries.
- [TypeScript Edition](/1.0.0-beta.2/extending/editions/typescript): TypeScript. Node, Bun, Deno, frontend tooling.
- [Python Edition](/1.0.0-beta.2/extending/editions/python): Python. Python packages, data tooling, scripts.

Not sure? Start with Dang. Use another SDK when you need that language's ecosystem.

## What SDK Pages Cover

Each SDK page owns its own setup:

- installing the SDK
- initializing a new module
- regenerating SDK-managed files
- adding dependencies
- IDE setup and project layout
