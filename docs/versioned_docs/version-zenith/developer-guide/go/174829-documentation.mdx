# Inline Documentation

<PartialExperimentalDocs />

To make your module easy to use for others, consider documenting its functions, objects and fields. This documentation is displayed when calling `dagger functions` and for the `--help` flag of `dagger call`.

## Module documentation

The following code snippet shows how to add high-level module documentation:

```go file=./snippets/documentation/module/main.go
```

## Object documentation

The following code snippet shows how to add documentation for an object of your module:

```go file=./snippets/documentation/object/main.go
```

## Function documentation

The following code snippet shows how to add documentation for a function in your module:

```go file=./snippets/documentation/function/main.go
```

Here is an example of the result from `dagger call add --help`:

```shell
Compute the sum of two numbers.

Example usage: dagger call add --a=4 --b=5

Usage:
  dagger call add [flags]

EXPERIMENTAL:
  dagger call add is currently under development and may change in the future.

Flags:
      --a int   The first number to add (default 4)
      --b int   The second number to add
```
