To make your module as simple to use as possible for others, it's a good practise to document it.  
There are multiple part of your module that you can document.

## Module documentation

The following code snippet shows how you can add a top level module documentation.  

```typescript file=./snippets/documentation/module/index.ts
```

## Object documentation

The following code snippet shows how you can add documentation for an object of your module.  

```typescript file=./snippets/documentation/object/index.ts
```

## Function documentation

The following code snippet shows how you can add document for a function in your object.

```typescript file=./snippets/documentation/function/index.ts
```

The documentation will be displayed when you use the `--help` flag:

```shell
dagger call add --help
```

And the result will be:

```shell
Compute the sum of two numbers.

Example usage: dagger call add --a 4 --b 5

Usage:
  dagger call add [flags]

EXPERIMENTAL:
  dagger call add is currently under development and may change in the future.

Flags:
      --a int   The first number to add (default 4)
      --b int   The second number to add    
```

The following code snippet shows how you can add document for a field in your object.
